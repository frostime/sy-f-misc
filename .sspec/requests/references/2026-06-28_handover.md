## Task Context

当前项目里的 Zotero 功能位于 `src/func/zotero/`，其核心做法是把 JS 代码 POST 到 `http://127.0.0.1:23119/debug-bridge/execute`，依赖 Better BibTeX 的 debug-bridge。

本次预调研的核心问题是：

1. 哪些能力可以直接改用 Zotero 官方 Local API；
2. 哪些能力仍然需要一个运行在 Zotero 内部的扩展来桥接；
3. 这种桥接扩展是否能做成一个足够小、足够稳定的原型。

结论是：**可以把“数据访问”与“UI 状态访问”拆开**：

- 数据访问（条目、子项、集合、搜索等）→ 用官方 Local API；
- UI 状态访问（尤其是“当前选中的文献项”）→ 用最小 Zotero 扩展暴露一个本地 HTTP 端点。

本 handover 只保留后续优化所需的结论、状态和文件入口，不保留本 session 中间的大量失败尝试。

## Completed Work

- Zotero Local API 基址：`http://127.0.0.1:23119/api/`

### 3. 原型优先级

本次 bridge 的目标不是做完整 Zotero 插件体系，而是做**尽量小的原型**，验证技术路径。
