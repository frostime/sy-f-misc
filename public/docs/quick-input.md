# QuickInput 使用说明

QuickInput 是一个可配置的快速插入面板。按下 `Alt + I` 呼出面板，点击预设即可在指定位置自动创建文档或插入块。

## 基本流程

1. 在设置 → ✅ 启用功能中打开 `💡 QuickInput`。
2. 进入设置 → 💡 QuickInput，新增一个模板。
3. 配置「插入位置」和「内容模板」。
4. 按 `Alt + I` 呼出面板，点击模板名称执行。

---

## 基础信息

| 字段 | 含义 |
|------|------|
| 名称 | 面板上显示的按钮文字。 |
| 图标 | 按钮前显示的 emoji 或符号。 |
| 分组 | 用于在面板上分组展示。 |
| 插入后打开 | 执行后是否自动跳转到新创建的文档/块。 |

## 插入位置

### 类型：新建文档

在指定笔记本的指定 hpath 下创建新文档。

| 字段 | 含义 | 示例 |
|------|------|------|
| 笔记本 ID | 目标笔记本 ID。 | `20231224140619-bpyuay4` |
| 文档 hpath | 文档路径，支持 `${var}` 插值。 | `/QuickInput/${date}-${title}` |

如果 hpath 中的父路径不存在，思源会自动创建中间文档。

### 类型：插入块

将内容插入到某个已有块（anchorId）的相对位置。

| 字段 | 含义 | 示例 |
|------|------|------|
| anchorId | 目标块 ID，支持 `${var}` 插值。 | `20260625020808-5euh5ov` 或 `${todayDailynoteId}` |
| 插入方式 | 相对 anchorId 的位置。 | 见下方 |
| 日记本 ID | 当 anchorId 引用 `${todayDailynoteId}` 时必填。 | `20231224140619-bpyuay4` |

#### 插入方式

| 方式 | 含义 |
|------|------|
| 作为末尾子块 | 插入到 anchorId 内部的最后。 |
| 作为首个子块 | 插入到 anchorId 内部的最前。 |
| 作为前一个同级块 | 插入到 anchorId 的前一个兄弟位置。 |
| 作为后一个同级块 | 插入到 anchorId 的后一个兄弟位置。 |

#### 今日日记快捷填充

点击「插入今日日记」按钮，会自动把 anchorId 填为 `${todayDailynoteId}`、插入方式设为「作为末尾子块」，并提示你填写日记本 ID。

也可以直接在 anchorId 中使用内联语法指定日记本：

```text
${todayDailynoteId:20231224140619-bpyuay4}
```

这会优先于「日记本 ID」字段。

## 内容模板

内容模板就是最终要插入的 Markdown 文本，支持 `${var}` 插值。

### 内置变量

| 变量 | 含义 | 示例 |
|------|------|------|
| `${year}` | 当前年份 | `2026` |
| `${month}` | 当前月份（补零） | `06` |
| `${day}` | 当前日期（补零） | `25` |
| `${hour}` | 当前小时（补零） | `15` |
| `${minute}` | 当前分钟（补零） | `44` |
| `${second}` | 当前秒（补零） | `56` |
| `${date}` | 当前日期 | `2026-06-25` |
| `${time}` | 当前时间 | `15:44:56` |
| `${datetime}` | 当前日期时间 | `2026-06-25 15:44:56` |
| `${todayDailynoteId}` | 今日日记文档 ID | `20260625020834-76ieh91` |

### 用户输入变量

在「输入字段」区域声明的字段，会成为执行时弹出的表单。字段的 `key` 就是模板中可用的变量名。

例如声明一个 `title` 字段，模板中写：

```markdown
# ${title}
```

执行时就会弹出输入框让你填写 `title`。

### 块属性（IAL）

思源支持 kramdown 内联属性列表。你可以在模板末尾写：

```markdown
# ${title}
{: custom-type="quick-input" custom-status="${status}"}
```

内核会把 `custom-*` 属性挂到新块上。

### 转义

如果不想让 `${var}` 被解析，写成 `\${var}`，最终输出会保留为 `${var}`。

## 输入字段

每个字段会映射为执行时弹出的表单控件。

| 字段属性 | 含义 |
|----------|------|
| key | 变量名，模板中通过 `${key}` 引用。必填。 |
| label | 表单上显示的标签。 |
| type | 控件类型：文本、多行文本、数字、开关、下拉选择。 |
| value | 默认值。 |
| placeholder | 输入框占位提示。 |

下拉选择类型需要在默认值编辑器下方的文本框里填写选项，每行一个，格式为 `key=显示文字` 或只写 `key`。

## 完整示例

### 示例 1：每日 Issue 文档

基础信息：
- 名称：今日 Issue
- 分组：工作

插入位置：
- 类型：新建文档
- 笔记本 ID：`你的笔记本 ID`
- 文档 hpath：`/Issues/${date}-${title}`

输入字段：
- key：`title`，label：`标题`，type：文本

内容模板：

```markdown
# ${title}
创建时间：${datetime}
{: custom-type="issue" custom-status="open"}
```

### 示例 2：闪念插入今日日记

基础信息：
- 名称：闪念
- 分组：日记

插入位置：
- 类型：插入块
- anchorId：`${todayDailynoteId:你的日记本ID}`
- 插入方式：作为末尾子块

输入字段：
- key：`idea`，label：`闪念`，type：多行文本

内容模板：

```markdown
- ${datetime}：${idea}
```

## 常见问题

**Q：创建文档时同一路径已存在会怎样？**  
A：QuickInput 会提示你已存在多少个同名文档，确认后才创建，避免静默重复。

**Q：block 模式提示「目标块不存在」？**  
A：请检查 anchorId 是否填写正确。如果使用了 `${todayDailynoteId}`，请确保填写了日记本 ID。

**Q：变量没有替换成功？**  
A：检查变量名是否与输入字段的 `key` 完全一致，且是否写成了 `${key}` 形式。
